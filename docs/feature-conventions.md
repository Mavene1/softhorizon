# Page & Section Conventions

> **This doc has been superseded by `docs/pages.md`.**  
> See `docs/pages.md` for the current route map, section structure, naming conventions, and Server vs Client Component decision rules.

The older "feature folder" pattern from a SaaS template does not apply here. Pages are organized by route (`app/about/`, `app/projects/[slug]/`, etc.) and sections by page (`components/sections/home/`, `components/sections/team/`, etc.).

For all routing, section structure, and naming conventions, read `docs/pages.md`.
